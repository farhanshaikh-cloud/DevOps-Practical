Based on your command history, you've practiced the fundamental Docker image and container operations. Here's a clean explanation of each command sequence.

| Command                                      | Description                                                              |
| -------------------------------------------- | ------------------------------------------------------------------------ |
| `docker --version`                           | Displays the installed Docker version.                                   |
| `docker images`                              | Lists all Docker images stored locally.                                  |
| `clear`                                      | Clears the terminal screen.                                              |
| `docker pull nginx:latest`                   | Downloads the latest Nginx image from Docker Hub.                        |
| `docker run nginx`                           | Creates and starts an Nginx container in the foreground.                 |
| `docker run -d nginx`                        | Starts an Nginx container in detached (background) mode.                 |
| `docker logs <container_id>`                 | Displays logs generated by the specified container.                      |
| `docker container ls`                        | Shows all running containers.                                            |
| `docker run -d -p 80:80 nginx`               | Runs Nginx in the background and maps host port 80 to container port 80. |
| `docker container ls -a` *(you typed `-la`)* | Lists all containers, including stopped ones.                            |
| `docker run -d -p 1000:80 nginx`             | Runs another Nginx container accessible on `http://<server-ip>:1000`.    |
| `docker run -d -p 2000:80 nginx:1.31`        | Runs a specific version (`1.31`) of Nginx on host port 2000.             |
| `docker images`                              | Displays all downloaded images.                                          |
| `docker ps -a`                               | Shows all containers (running and stopped).                              |
| `docker container stop <container_id>`       | Stops a running container.                                               |
| `docker container start <container_id>`      | Starts a stopped container.                                              |
| `docker container rm <container_id>`         | Removes a stopped container.                                             |
| `docker rmi <image_id>`                      | Deletes an image using its Image ID.                                     |
| `docker rmi nginx:1.31`                      | Removes the Nginx 1.31 image.                                            |
| `docker rmi nginx:latest`                    | Removes the latest Nginx image.                                          |
| `history`                                    | Displays the shell command history.                                      |

### Common Docker commands cheat sheet

```bash
# Check Docker version
docker --version

# Download an image
docker pull nginx

# List images
docker images

# Run a container
docker run -d -p 80:80 nginx

# List running containers
docker ps

# List all containers
docker ps -a

# View logs
docker logs <container_id>

# Stop container
docker stop <container_id>

# Start container
docker start <container_id>

# Remove container
docker rm <container_id>

# Remove image
docker rmi <image_name_or_image_id>

# Remove all stopped containers
docker container prune

# Remove unused images
docker image prune -a
```
